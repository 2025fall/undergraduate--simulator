## 1. 产品概述 (Product Overview)

### 1.1 背景与目标
基于当前互联网求职环境与大学生现状，开发一款具有黑色幽默风格的文字模拟养成游戏。旨在通过夸张与拟真的手法，让玩家体验从大一入学到毕业求职的四年历程，体验资源管理与人生抉择的乐趣。

### 1.2 核心概念
- 模拟对象：本科四年全周期
- 核心冲突：有限的时间/精力 vs 无限的内卷要求（GPA、实习、考研）
- 核心玩法：Roguelike开局 + 资源管理 + 简历养成 + 回合制面试

### 1.3 目标用户
- 在校大学生（共鸣、预演）
- 互联网从业者（玩梗、怀旧）
- 模拟经营游戏爱好者

## 2. 角色系统 (Character System)

### 2.1 随机开局 (Roguelike)
游戏开始时，系统随机生成5名角色供玩家选择（N选1）。

#### 属性维度
**学校背景** (影响初始属性、简历通过率基础值)：
| 学校类型 | 特点 |
|---------|------|
| Top2 | 初始属性高，简历通过率极高 |
| 985 | 初始属性较高，简历通过率高 |
| 211 | 初始属性中等 |
| 双非 | 初始属性一般，简历筛选有惩罚 |
| 民办 | 初始属性较低，简历筛选困难模式 |

**家庭背景** (提供特殊Buff)：
| 家庭类型 | Buff效果 |
|---------|---------|
| 富二代 | 心态恢复速度 +50%（失败惩罚减半） |
| 中产家庭 | 初始软技能 +10 |
| 互联网世家 | 大三30%概率触发"亲戚内推"事件 |
| 工薪阶层 | 精力上限 +20（由100变为120） |

**基础三维** (0-999)：
- 项目分 (Project)：决定简历筛选率
- 八股分 (Knowledge)：决定技术面通过率
- 软技能 (SoftSkill)：决定HR面通过率

## 3. 核心循环与时间轴 (Core Loop & Timeline)

### 3.1 时间系统
- 总时长：48个月（4年）
- 推进方式：
  - 手动结束：点击"结束本月"，时间+1月
  - 事件跳过：实习跳过3个月

### 3.2 阶段划分
| 阶段 | 时间范围 | 特点 |
|------|---------|------|
| 积累期 | 第1-24月 | 刷GPA，积累基础属性 |
| 实习期 | 第25-36月 | 找实习，丰富简历 |
| 抉择期 | 第37-48月 | 秋招/考研冲刺 |

### 3.3 资源系统
| 资源 | 初始值 | 作用 |
|------|-------|------|
| 精力 | 100 | 执行行动消耗 |
| 心态 | 80 | 低于20限制高难度行动 |
| GPA | 3.5 | <2.0触发延毕 |

## 4. 功能模块详解

### 4.1 基础行动
| 行动 | 消耗精力 | 效果 |
|------|---------|------|
| 上课 | 20 | GPA +0.1 |
| 做项目 | 30 | 项目+8 |
| 刷题 | 20 | 八股+8 |
| 社交 | 20 | 软技能+8 |

### 4.2 面试系统
- 流程：简历筛选 -> 技术面 -> HR面
- 公司分级：T1(大厂)到T3(小厂)

## 5. 结局系统
| 结局 | 触发条件 |
|------|---------|
